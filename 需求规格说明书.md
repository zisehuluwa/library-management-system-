## 一、项目概述

### 1.1 项目背景

随着图书馆藏书量的不断增长和读者规模的扩大，传统的人工管理模式已难以满足业务需求。本系统旨在通过数字化手段，实现图书馆日常业务的全面管理，提升工作效率，改善读者服务体验。

### 1.2 项目目标

- 实现图书、读者、流通业务的数字化管理
- 提高图书借还效率，减少人工错误
- 提供便捷的图书检索与预约服务
- 实现超期提醒与违约金自动计算
- 为管理者提供多维度的数据统计分析

### 1.3 目标用户

| 用户角色 | 描述 |
|---------|------|
| 系统管理员 | 负责系统配置、用户管理、数据维护 |
| 图书馆员 | 负责图书编目、借还办理、读者服务 |
| 读者 | 借阅图书、查询书目、预约续借 |
| 馆长/管理层 | 查看统计报表、辅助决策 |

### 1.4 术语定义

- **ISBN**：国际标准书号（International Standard Book Number）
- **流通**：图书在馆员与读者之间的借出与归还流转过程
- **续借**：读者在图书到期前申请延长借阅期限
- **超期违约金**：读者未在规定期限内归还图书产生的罚款

## 二、功能需求

### 2.1 图书管理

#### 2.1.1 图书编目

系统管理员和图书馆员可对图书进行编目入库，字段包括：

| 字段 | 说明 | 必填 |
|------|------|------|
| ISBN | 国际标准书号 | 是 |
| 书名 | 图书名称 | 是 |
| 作者 | 著者信息 | 是 |
| 出版社 | 出版机构 | 是 |
| 出版日期 | 出版时间 | 是 |
| 分类 | 依据《中国图书馆分类法》 | 是 |
| 价格 | 图书定价 | 是 |
| 馆藏数量 | 购买册数 | 是 |
| 存放位置 | 书架编号/区域 | 是 |
| 封面图片 | 图书封面 | 否 |
| 内容简介 | 图书描述 | 否 |

#### 2.1.2 图书查询

- 支持按书名、作者、ISBN、分类、出版社等多维度检索
- 支持模糊搜索
- 支持按出版年份、价格区间筛选
- 显示图书当前状态（在馆/已借出/已预约/下架）

#### 2.1.3 图书维护

- 修改图书信息
- 图书下架/注销（注明原因）
- 图书遗失登记
- 图书丢失赔偿处理

---

### 2.2 读者管理

#### 2.2.1 读者注册

| 字段 | 说明 | 必填 |
|------|------|------|
| 姓名 | 真实姓名 | 是 |
| 性别 | 男/女 | 是 |
| 证件类型 | 身份证/护照/学生证 | 是 |
| 证件号码 | 证件编号 | 是 |
| 联系电话 | 手机号码 | 是 |
| 邮箱 | 电子邮箱 | 否 |
| 地址 | 通讯地址 | 否 |
| 读者类型 | 普通读者/VIP读者/教师/学生 | 是 |
| 押金 | 借阅押金（根据读者类型） | 是 |

#### 2.2.2 读者证管理

- 自动生成读者证号（规则：年月日内4位序号）
- 读者证状态：正常/挂失/冻结/注销
- 读者证有效期管理（可按类型设置不同有效期）
- 读者证密码设置（用于自助查询/登录）

#### 2.2.3 读者查询与维护

- 按姓名、证件号、读者证号查询
- 修改读者信息
- 读者证挂失/解挂
- 读者证注销
- 冻结/解冻（超期未还、欠款等）

---

### 2.3 借阅流通管理

#### 2.3.1 借书

- 扫描读者证确认身份
- 扫描图书条码完成借阅
- 自动检查：
  - 读者证是否有效
  - 是否有超期未还图书
  - 是否有欠款
  - 借阅数量是否达到上限
- 记录借阅信息（借书日期、应还日期、操作员）
- 打印借书凭证

**借阅规则（默认配置，可调整）：**

| 读者类型 | 借阅数量上限 | 借阅天数 | 续借次数 | 超期违约金(元/天/册) |
|---------|------------|--------|---------|-------------------|
| 普通读者 | 5册 | 30天 | 1次 | 0.10 |
| VIP读者 | 10册 | 60天 | 2次 | 0.05 |
| 教师 | 20册 | 180天 | 3次 | 0 |
| 学生 | 10册 | 60天 | 2次 | 0.05 |

#### 2.3.2 还书

- 扫描图书条码完成归还
- 自动计算是否超期
- 超期自动计算违约金
- 打印还书凭证（含超期信息）
- 读者欠款登记

#### 2.3.3 续借

- 读者可通过系统自助续借或柜台办理
- 仅限未超期图书
- 续借后借期顺延（按原类型规则）
- 每个读者类型限制续借次数

#### 2.3.4 预约

- 读者可预约已借出的图书
- 预约到货后通知读者
- 预约保留期限（默认3天）
- 取消预约

---

### 2.4 馆藏管理

#### 2.4.1 馆藏统计

- 按分类统计馆藏数量与比例
- 按年份统计新增图书
- 按馆藏状态统计（在馆/借出/遗失/下架）
- 按出版社统计

#### 2.4.2 馆藏盘点

- 生成盘点清单
- 记录实盘数据
- 生成损溢报表

---

### 2.5 超期与违约金管理

#### 2.5.1 超期提醒

- 借阅到期前3天发送提醒（短信/系统通知）
- 借阅到期当天发送提醒
- 超期后每日发送催还通知

#### 2.5.2 违约金处理

- 自动计算超期违约金
- 读者缴纳违约金
- 违约金缴纳记录
- 欠款查询与催缴

---

### 2.6 统计分析

#### 2.6.1 流通统计

- 日/周/月/年借还量统计
- 高借阅量图书排行
- 借阅高峰时段分析

#### 2.6.2 读者统计

- 新增读者统计
- 读者活跃度分析
- 读者流失预警

#### 2.6.3 报表导出

- 支持导出为Excel、PDF格式
- 自定义时间范围
- 自定义统计维度

---

### 2.7 系统管理

#### 2.7.1 用户与权限

- 系统操作用户管理（图书馆员、管理员）
- 角色与权限配置
- 操作日志记录与查询

#### 2.7.2 参数配置

- 借阅规则配置
- 违约金费率配置
- 预约规则配置
- 通知模板配置

#### 2.7.3 数据管理

- 数据备份
- 数据恢复
- 操作日志

---

## 三、非功能需求

### 3.1 性能需求

- 系统支持 100 人同时在线操作
- 图书检索响应时间 ≤ 2秒
- 借还操作响应时间 ≤ 1秒

### 3.2 安全性需求

- 用户密码加密存储
- 操作权限严格控制
- 敏感操作日志审计
- 数据定期备份

### 3.3 可用性需求

- 界面简洁直观，操作便捷
- 提供操作指引和帮助文档
- 系统故障恢复时间 ≤ 4小时

### 3.4 扩展性需求

- 支持后续接入自助借还机
- 支持接入电子图书系统
- 支持多图书馆分馆管理

## 五、需求优先级

| 优先级 | 功能模块 |
|--------|---------|
| P0（核心） | 图书管理、读者管理、借还流通 |
| P1（重要） | 超期管理、统计分析、预约续借 |
| P2（增强） | 馆藏盘点、系统管理、数据导出 |

## 六、验收标准

1. ✅ 图书可正常完成增删改查操作
2. ✅ 读者可注册并办理借阅证
3. ✅ 借还流程完整，数据准确
4. ✅ 超期违约金计算正确
5. ✅ 统计分析报表可正常生成
6. ✅ 系统操作有完整日志记录
